---
title: 'Update Application Status'
api: 'PATCH /api/v1/companies/me/applications/:id'
description: 'Update the status of a job application'
---

## Endpoint

```
PATCH /api/v1/companies/me/applications/:id
```

**Base URL**: `http://localhost:3000/api/v1`

<Note>This endpoint requires **Company authentication**.</Note>

## Headers

| Header        | Value                 |
| ------------- | --------------------- |
| Authorization | Bearer {access_token} |

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the application (UUID).
</ParamField>

## Request Body

<ParamField body="status" type="enum" required>
  The new status for the application. Supported values:

  - `PENDING`
  - `REVIEWED`
  - `SHORTLISTED`
  - `INTERVIEW_SCHEDULED`
  - `REJECTED`
  - `HIRED`
  - `WITHDRAWN`
</ParamField>

## Example Request

```json
{
  "status": "SHORTLISTED"
}
```

## Response

### Success Response (200 OK)

```json
{
  "success": true,
  "message": "Application status updated successfully",
  "meta": {
    "timestamp": "2026-03-07T21:15:00.000Z",
    "path": "/companies/me/applications/5d524fd9-8ea1-4c4f-bc15-b32072c144a1",
    "method": "PATCH"
  }
}
```

<Info>
  **Current Implementation**: This endpoint currently returns a success envelope only. It does not
  return the updated application object.
</Info>
